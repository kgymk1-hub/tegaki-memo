# tegaki-memo（手書きメモ）

ブラウザだけで動く、シンプルな手書きメモ・お絵かきアプリです。HTML / CSS / JavaScript の静的ファイルだけで構成しているため、GitHub Pages で公開できます。スマホ・タブレット・PC から、指・タッチペン・マウスでメモやラフスケッチを作成できます。

## GitHub Pages URL

```text
https://kgymk1-hub.github.io/tegaki-memo/
```

## 実装済み機能

- canvas への手書き描画
- ペン / 半透明マーカー / 消しゴム
- 色変更（プリセット色・任意色）
- 太さ変更
- 図形・注釈ツール
  - 直線（選択後、キャンバス上でドラッグして描画）
  - 四角形（選択後、キャンバス上でドラッグして描画）
  - 円 / 楕円（選択後、キャンバス上でドラッグして描画）
  - 矢印（選択後、キャンバス上でドラッグして描画）
  - 点線（選択後、キャンバス上でドラッグして描画）
  - テキスト入力（選択後、文字を入れたい位置をタップ。入力後・キャンセル後・空欄入力後は自動でペンへ戻る）
- Undo
  - 描画（ペン / マーカー / 図形 / テキスト）
  - 現在レイヤーの消去
  - 画像読込
  - レイヤー追加 / 複製 / 削除 / 名前変更 / 表示・非表示 / 順序変更 / 下へ結合
  - 背景モード変更 / 背景色変更
  - レイヤー透明度変更
- 背景設定
  - 白
  - 任意色
  - 透明（画面上はチェック柄）
- 画像読込
  - 新規レイヤーとして読み込み
  - 現在レイヤーへ読み込み
  - 読込直後に中央配置のプレビューを表示
  - 配置中はドラッグで位置調整
  - 「確定」でレイヤーへ反映、「取消」で読込をキャンセル
  - 配置中は通常描画やPNG保存の前に確定 / 取消が必要
- PNG保存
  - 表示中レイヤーのみを書き出し
  - 背景設定とレイヤー透明度を反映
- レイヤー管理
  - 最大5枚
  - 非表示レイヤーには描画・文字入力・現在レイヤーへの画像読込はできず、操作時に案内を表示
  - 追加 / 複製 / 削除
  - 複製は表示中レイヤーのみ可能（非表示レイヤーは複製不可）
  - レイヤー名変更
  - 表示 / 非表示
  - 上下移動
  - 下へ結合
  - 透明度調整
- PWA対応
  - `manifest.json`
  - `service-worker.js`
  - ホーム画面追加対応
  - アプリシェルのキャッシュ対応

## 使い方

1. GitHub Pages URL、またはローカルの `index.html` をブラウザで開きます。
2. キャンバス上を指・タッチペン・マウスでなぞると描画できます。
3. ツールバーでペン、マーカー、消しゴム、図形、文字、色、太さ、背景、レイヤーを切り替えます。
4. 直線・四角形・円 / 楕円・矢印・点線は、ツールを選んだあとキャンバス上でドラッグして描画します。
5. 文字ツールは、ツールを選んだあと文字を入れたい位置をタップして入力します。入力後、キャンセル後、空欄入力後はいずれも自動でペンへ戻ります。
6. 「選択レイヤー消去」はアプリ全体ではなく、選択中レイヤーだけを消去します。
7. 非表示レイヤーには描画できません。非表示レイヤーを選択中に描画・文字入力・現在レイヤーへの画像読込をしようとすると、表示に切り替えるか別レイヤーを選ぶよう案内されます。
8. 非表示レイヤーは複製できません。表示中のレイヤーを選択してから複製してください。
9. 「画像読込先」で画像を「新規レイヤー」または「現在レイヤー」に読み込むか選び、「画像読込」ボタンから画像ファイルを選択します。
10. 画像を読み込むとキャンバス中央にプレビュー表示されます。ドラッグで位置を調整し、「確定」でレイヤーへ反映するか、「取消」で読込をキャンセルします。
11. 画像配置中はペン・図形・文字などの通常描画は行えません。PNG保存や再度の画像読込を行う前に、画像を確定または取消してください。
12. レイヤーの「選択レイヤー透明度」スライダーで選択中レイヤーの見え方を調整できます。
13. 「戻す」で直前の操作をUndoできます。
14. 「PNG保存」で現在の表示内容を画像として保存できます。

## ファイル構成

```text
tegaki-memo/
├─ index.html         # 画面構造、PWA manifest / theme-color 読み込み
├─ style.css          # レイアウト・見た目・スマホ向け調整
├─ app.js             # 描画処理・Undo・背景・画像読込・レイヤー管理・PNG保存・service worker 登録
├─ manifest.json      # PWA設定
├─ service-worker.js  # キャッシュ制御
└─ icons/
   └─ icon.svg        # PWAアイコン
```

## PWAとしての使い方

- GitHub Pages URLをブラウザで開き、ブラウザの「ホーム画面に追加」または「アプリをインストール」から追加できます。
- `manifest.json` と `service-worker.js` は相対パスで読み込むため、GitHub Pages のサブディレクトリ配信でも動作します。
- service worker はアプリシェル（HTML / CSS / JavaScript / manifest / SVGアイコン）をキャッシュするため、一度開いた後はオフラインでも起動しやすくなります。
- 更新反映を安定させるため、HTML / CSS / JavaScript はネットワーク優先で取得し、取得に成功した場合はキャッシュを更新します。ネットワークに接続できない場合のみキャッシュを利用します。
- PWAやGitHub Pagesでは、デプロイ後もしばらく古いHTML / CSS / JavaScriptがキャッシュから表示される場合があります。

### 更新が反映されない場合

GitHub PagesやPWAでは、Service Workerやブラウザキャッシュにより古い `app.js` / `style.css` / `index.html` が残る場合があります。更新が反映されない場合は、以下を試してください。

1. ページを数回再読み込みする
2. Chromeのサイトデータ / キャッシュを削除する
3. ホーム画面に追加したPWAを一度削除する
4. `service-worker.js` の `CACHE_NAME` が更新されているか確認する
5. 再度GitHub Pagesを開き、必要に応じてホーム画面に追加し直す

開発者向けの注意: `app.js` / `style.css` / `index.html` を変更した場合は、`service-worker.js` の `CACHE_NAME` も更新してください。キャッシュ対象のクエリ文字列（例: `app.js?v=8`）を使っている場合は、`index.html` と `service-worker.js` の両方で同じ版にそろえてください。

## 実装方針メモ

- 既存の操作感を大きく変えず、ツールバーは機能ごとに行を分けています。
- Undo履歴にはレイヤー状態、アクティブレイヤー、ヒント表示状態、背景設定を保存しています。
- 「選択レイヤー消去」は選択中レイヤーだけを消去する操作として扱い、表示中レイヤーに描画が残っている場合は初期ヒントを再表示しないようにしています。
- 描画可否判定は共通化し、非表示レイヤー選択中の描画・文字入力・現在レイヤーへの画像読込では案内を表示して、不要なUndo履歴を積まないようにしています。
- ヒント表示はツール種別とアクティブレイヤーの表示状態に合わせ、図形はドラッグ、文字はタップ、非表示レイヤーは描画不可が伝わるようにしています。
- 下へ結合は非表示レイヤーが絡む場合に禁止し、見た目と結果がずれないようにしています。
- 画像読込は「新規レイヤー」を初期値にしつつ、「現在レイヤー」へ読み込む選択肢も残しています。
- 画像読込後はすぐにレイヤーへ焼き付けず、表示用canvas上のプレビューとして配置調整します。確定時だけUndo履歴を1回積み、取消時やドラッグ移動中は履歴を増やしません。
- 画像配置中は通常描画・PNG保存・再度の画像読込を止め、確定または取消を促します。
