# Tegaki Memo

ブラウザで動く手書きメモ・簡易お絵かきPWAです。
HTML / CSS / JavaScript の静的ファイルのみで作成しており、GitHub Pagesで公開できます。

## 概要

Tegaki Memoは、スマホ・タブレット・PCのブラウザ上で、指・タッチペン・マウスを使ってメモやラフスケッチを作成できるPWAです。ペン描画、図形、文字、レイヤー、画像配置、選択範囲操作、PNG保存、再編集用 `.tegaki` 保存に対応しています。

## 公開URL

```text
https://kgymk1-hub.github.io/tegaki-memo/
```

## 主な機能

- ペン、半透明マーカー、消しゴム
- 直線、四角形、円 / 楕円、矢印、点線
- 文字入力（入力後はペンへ戻ります）
- Undo
- 背景設定（白 / 任意色 / 透明）
- 画像読込、配置、ドラッグ移動、専用バーでの拡大縮小、左90° / 右90°回転、確定 / 取消
- レイヤー追加 / 削除 / 複製 / 名前変更 / 表示切替 / 順序変更 / 下へ結合 / 透明度調整
- 矩形選択、コピー、切り取り、貼り付け
- PNG保存
- 自動保存、前回作業復元、自動保存データ削除
- 独自形式 `.tegaki` 保存 / 読込
- キャンバスサイズ変更
- ピンチズーム
- 基本ツール行、クイックカテゴリ行、クイックパネル行によるスマホ向けUI
- PWA対応、Network FirstのService Worker

## 使い方

### UI

- スマホで描画エリアを広く使えるよう、上部に基本ツール行とクイックカテゴリ行を配置しています。
- 基本ツール行には、ペン、マーカー、消しゴム、戻す、PNG保存、メニューを配置しています。
- その下に、追加ツール、色・太さ、選択操作、レイヤー操作のクイックカテゴリを配置しています。カテゴリを押すと、対応する下位ツールだけが横一列で表示されます。
- 詳細メニューには、保存 / 読込、画像操作、設定、Aboutなど、使用頻度が低めの操作をまとめています。
- スマホでキャンバス下部まで操作しやすいよう、実表示領域に合わせた高さ制御と下部の安全余白を確保しています。
- 画像配置中は、キャンバスを見ながら確定・取消・倍率変更・90°回転ができる専用バーを画面下部に表示します。

### 基本描画

1. 公開URLまたはローカルの `index.html` をブラウザで開きます。
2. ペン、マーカー、消しゴムを選び、キャンバス上をドラッグして描画します。
3. プリセット色、任意色、太さは、上部の「色・太さ」クイックカテゴリから変更できます。
4. 「戻す」で直前の操作をUndoできます。

### 図形・文字

- 直線、四角形、円 / 楕円、矢印、点線は、ツール選択後にキャンバス上をドラッグして描画します。
- 文字ツールは、文字を入れたい位置をタップして入力します。入力後、キャンセル後、空欄入力後はいずれも自動でペンに戻ります。

### レイヤー

- レイヤーの追加、複製、削除、表示切替、順序変更、結合、透明度変更、レイヤー消去は「レイヤー操作」クイックカテゴリから操作できます。名前変更などの詳細操作も利用できます。
- 非表示レイヤーには描画できません。非表示レイヤー選択中はalertではなく画面上のヒントで案内します。
- 「選択レイヤー消去」は、選択中レイヤーだけを消去します。

### 画像配置

1. 「画像読込先」で「新規レイヤー」または「現在レイヤーを置換」を選びます。「現在レイヤーを置換」は現在レイヤーの内容を消して画像を配置するため、既存メモを残したい場合は「新規レイヤー」を選んでください。
2. 「画像読込」から画像ファイルを選択します。読込後は詳細メニューが閉じ、画面下部に画像配置専用バーが表示されます。
3. プレビューをドラッグして移動し、専用バーの倍率スライダーで拡大縮小できます。
4. 専用バーの左90° / 右90°ボタンで90度単位の回転ができます。
5. 専用バーの「確定」でレイヤーへ反映し、「取消」で破棄します。

### 選択範囲

- 選択、コピー、切り取り、貼り付け、解除は「選択操作」クイックカテゴリから操作できます。
- 「選択」ツールでキャンバスをドラッグすると、現在レイヤー上に矩形選択を作成できます。
- 貼り付け後はプレビューをドラッグ移動し、「確定」で反映、「取消」で破棄します。

### 保存・読込

- 「PNG保存」で完成画像を書き出します。
- 「作業保存」で再編集用 `.tegaki` ファイルを保存します。
- 「作業読込」で `.tegaki` またはJSONファイルを読み込み、現在の作業内容を置き換えます。

### キャンバスサイズ変更

- 「メニュー」内の設定で、幅・高さ入力またはプリセットからキャンバスサイズを変更できます。
- サイズ変更時、既存内容は左上基準で保持されます。

### 表示倍率

- 2本指ピンチで表示倍率を変更できます。
- 表示倍率は5%〜400%の範囲で、数値入力でも指定できます。
- 「全体表示」により、大きなキャンバスも画面内に収めて確認できます。
- 「等倍」で100%表示に戻せます。

### 自動保存と前回作業復元

- 描画やレイヤー操作などの変更後に、ブラウザ内へ自動保存します。
- 次回起動時に前回作業データがある場合、復元確認が表示されます。
- 「自動保存削除」で、ブラウザ内の復元用データだけを削除できます。

## 保存形式について

### PNG保存

完成画像として保存します。表示中レイヤー、背景設定、レイヤー透明度を反映します。PNGにはレイヤー情報は保持されません。

### .tegaki保存

レイヤー、背景、キャンバスサイズなどを保持した再編集用ファイルです。`.tegaki` ファイルはJSON形式で、レイヤー画像を含むためサイズが大きくなる場合があります。現在の選択状態や内部クリップボードは保存対象外です。

### 自動保存

ブラウザ内に保存される復元用データです。同じ端末・同じブラウザ・同じサイトデータ内で利用できます。ブラウザのサイトデータ削除により消える場合があります。

### 空判定の軽量化

表示中レイヤーが空かどうかの判定は、各レイヤーの `hasContent` フラグを使って軽量化しています。これにより、大きなキャンバスでも頻繁な全ピクセル走査を避けるようにしています。`.tegaki` 読込、自動保存復元、Undo、切り取り、結合、キャンバスサイズ変更など、内容が空に戻る可能性がある処理では必要に応じてキャンバス内容を再判定します。

## 注意事項

- 自動保存はブラウザ内のIndexedDBに保存されます。
- IndexedDBはlocalStorageより容量面で有利なため、大きなキャンバスや複数レイヤーの自動保存が安定しやすくなります。
- ブラウザのサイトデータ削除で自動保存データは消えます。
- `.tegaki` ファイルは、手動でファイルとして残す再編集用データです。自動保存とは別の仕組みです。
- 重要なデータは自動保存だけに頼らず、`.tegaki` 保存も併用してください。
- PNGは完成画像であり、レイヤー情報は保持しません。
- `.tegaki` ファイルはJSON形式で、画像を含むためサイズが大きくなる場合があります。
- 現在の選択状態や内部クリップボードは `.tegaki` 保存対象外です。
- ピンチズームは表示倍率のみで、保存画像サイズは変わりません。
- ピンチズーム時にも描画可能なキャンバス本体とスクロール用余白を見分けやすいよう、キャンバス外の領域を薄いグレーで表示しています。
- キャンバスサイズを縮小すると、はみ出た部分は切れます。
- PWA更新が反映されない場合は、キャッシュ削除やPWA再インストールが必要な場合があります。
- PWA用アイコンとして `icons/icon-192.png` と `icons/icon-512.png` を使用します。既存の `icons/icon.svg` も補助的に残しています。
- 外部ライブラリは使用していません。

`index.html` ではES Modulesを使わず、グローバルスコープのJavaScriptを `defer` 付きで読み込みます。アプリ表示上のバージョンは `Version 1.0.0` です。一方、`index.html` や `service-worker.js` で使用している `vXX` の番号は、PWAキャッシュ更新用の内部番号です。

## PWA更新が反映されない場合

GitHub PagesやPWAでは、Service Workerやブラウザキャッシュにより古いHTML / CSS / JavaScript / README / manifestが残る場合があります。更新が反映されない場合は、以下を試してください。

1. ページを数回再読み込みする
2. ブラウザのサイトデータ / キャッシュを削除する
3. ホーム画面に追加したPWAを一度削除する
4. 再度公開URLを開く
5. 必要に応じてPWAを再インストールする

開発時にHTML / CSS / JavaScript / README / manifestを変更した場合は、`index.html` のCSS / JS読み込みクエリ、`service-worker.js` の `CACHE_NAME`、`APP_SHELL` を同じキャッシュ更新番号にそろえて更新してください。

## 今後の改善候補

### 技術改善候補

- IndexedDBによる大容量自動保存は導入済みです。今後は必要に応じて保存世代管理や使用容量の見える化を検討します。
- Undo履歴はキャンバス画像を保持するため、履歴数やキャンバスサイズによってメモリ使用量が増えます。将来的には差分保存やImageBitmap活用などで削減する余地があります。
- 描画更新の一部では `requestAnimationFrame` による `renderAllLayers` 呼び出しの間引きを導入済みです。今後も描画遅延やプレビュー崩れがない範囲を確認しながら、適用範囲を広げる余地があります。
- CSS変数化は導入済みです。将来的にはこれを土台に、ライト / ダークなどのテーマ切替を検討します。
- 現在のJavaScriptはグローバルスコープ前提の分割ファイル構成です。規模が大きくなった場合はES Modules化により、依存関係やグローバル変数を整理することを検討します。
- Service Workerとキャッシュ更新番号は手動運用のため、更新漏れを防ぐチェック手順やリリース時の運用ルール改善を検討します。

### 機能改善候補

- 自由選択
