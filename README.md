# Tegaki Memo

ブラウザで動く手書きメモ・簡易お絵かきPWAです。
HTML / CSS / JavaScript の静的ファイルのみで作成しており、GitHub Pagesで公開できます。

## 概要

Tegaki Memoは、スマホ・タブレット・PCのブラウザ上で、指・タッチペン・マウスを使ってメモやラフスケッチを作成できるPWAです。ペン描画、図形、文字、レイヤー、画像配置、選択範囲操作、PNG保存、再編集用 `.tegaki` 保存に対応しています。

## 公開URL

```text
https://kgymk1-hub.github.io/tegaki-memo/
```

## 主な機能

- ペン、半透明マーカー、消しゴム
- 直線、四角形、円 / 楕円、矢印、点線
- 文字入力（入力後はペンへ戻ります）
- Undo
- 背景設定（白 / 任意色 / 透明）
- 画像読込、配置、ドラッグ移動、専用バーでの拡大縮小、左90° / 右90°回転、確定 / 取消
- レイヤー追加 / 削除 / 複製 / 名前変更 / 表示切替 / 順序変更 / 下へ結合 / 透明度調整
- 矩形選択、コピー、切り取り、貼り付け
- PNG保存
- 自動保存、前回作業復元、自動保存データ削除
- 独自形式 `.tegaki` 保存 / 読込
- キャンバスサイズ変更
- ピンチズーム
- コンパクトヘッダーと詳細メニューUI
- PWA対応、Network FirstのService Worker

## 使い方

### UI

- スマホで描画エリアを広く使えるよう、よく使う操作のみを上部に表示し、詳細操作は「メニュー」内にまとめています。
- 通常表示にはペン、マーカー、消しゴム、戻す、PNG保存、メニューを配置しています。
- 図形、色と太さ、作業保存 / 読込、画像読込、選択操作、レイヤー操作、設定、Aboutは詳細メニューから操作できます。
- スマホでキャンバス下部まで操作しやすいよう、実表示領域に合わせた高さ制御と下部の安全余白を確保しています。
- 画像配置中は、キャンバスを見ながら確定・取消・倍率変更・90°回転ができる専用バーを画面下部に表示します。

### 基本描画

1. 公開URLまたはローカルの `index.html` をブラウザで開きます。
2. ペン、マーカー、消しゴムを選び、キャンバス上をドラッグして描画します。
3. プリセット色、任意色、太さは「メニュー」内の「色と太さ」から変更できます。
4. 「戻す」で直前の操作をUndoできます。

### 図形・文字

- 直線、四角形、円 / 楕円、矢印、点線は、ツール選択後にキャンバス上をドラッグして描画します。
- 文字ツールは、文字を入れたい位置をタップして入力します。入力後、キャンセル後、空欄入力後はいずれも自動でペンに戻ります。

### レイヤー

- レイヤーは追加、削除、複製、名前変更、表示 / 非表示、上下移動、下へ結合、透明度調整ができます。
- 非表示レイヤーには描画できません。非表示レイヤー選択中はalertではなく画面上のヒントで案内します。
- 「選択レイヤー消去」は、選択中レイヤーだけを消去します。

### 画像配置

1. 「画像読込先」で新規レイヤーまたは現在レイヤーを選びます。
2. 「画像読込」から画像ファイルを選択します。読込後は詳細メニューが閉じ、画面下部に画像配置専用バーが表示されます。
3. プレビューをドラッグして移動し、専用バーの倍率スライダーで拡大縮小できます。
4. 専用バーの左90° / 右90°ボタンで90度単位の回転ができます。
5. 専用バーの「確定」でレイヤーへ反映し、「取消」で破棄します。

### 選択範囲

- 「選択」ツールでキャンバスをドラッグすると、現在レイヤー上に矩形選択を作成できます。
- コピー、切り取り、貼り付け、解除に対応しています。
- 貼り付け後はプレビューをドラッグ移動し、「確定」で反映、「取消」で破棄します。

### 保存・読込

- 「PNG保存」で完成画像を書き出します。
- 「作業保存」で再編集用 `.tegaki` ファイルを保存します。
- 「作業読込」で `.tegaki` またはJSONファイルを読み込み、現在の作業内容を置き換えます。

### キャンバスサイズ変更

- 「メニュー」内の設定で、幅・高さ入力またはプリセットからキャンバスサイズを変更できます。
- サイズ変更時、既存内容は左上基準で保持されます。

### ピンチズーム

- 2本指ピンチで表示倍率を変更できます。
- 「等倍」で100%表示に戻せます。

### 自動保存と前回作業復元

- 描画やレイヤー操作などの変更後に、ブラウザ内へ自動保存します。
- 次回起動時に前回作業データがある場合、復元確認が表示されます。
- 「自動保存削除」で、ブラウザ内の復元用データだけを削除できます。

## 保存形式について

### PNG保存

完成画像として保存します。表示中レイヤー、背景設定、レイヤー透明度を反映します。PNGにはレイヤー情報は保持されません。

### .tegaki保存

レイヤー、背景、キャンバスサイズなどを保持した再編集用ファイルです。`.tegaki` ファイルはJSON形式で、レイヤー画像を含むためサイズが大きくなる場合があります。現在の選択状態や内部クリップボードは保存対象外です。

### 自動保存

ブラウザ内に保存される復元用データです。同じ端末・同じブラウザ・同じサイトデータ内で利用できます。ブラウザのサイトデータ削除により消える場合があります。

## 注意事項

- 自動保存はブラウザ内に保存されます。
- ブラウザのサイトデータ削除で自動保存は消える場合があります。
- `.tegaki` ファイルは再編集用の作業ファイルです。
- PNGは完成画像であり、レイヤー情報は保持しません。
- `.tegaki` ファイルはJSON形式で、画像を含むためサイズが大きくなる場合があります。
- 現在の選択状態や内部クリップボードは `.tegaki` 保存対象外です。
- ピンチズームは表示倍率のみで、保存画像サイズは変わりません。
- ピンチズーム時にも描画可能なキャンバス本体とスクロール用余白を見分けやすいよう、キャンバス外の領域を薄いグレーで表示しています。
- キャンバスサイズを縮小すると、はみ出た部分は切れます。
- PWA更新が反映されない場合は、キャッシュ削除やPWA再インストールが必要な場合があります。
- PWA用アイコンとして `icons/icon-192.png` と `icons/icon-512.png` を使用します。既存の `icons/icon.svg` も補助的に残しています。
- 外部ライブラリは使用していません。

```text
tegaki-memo/
├─ index.html
├─ style.css
├─ manifest.json

├─ LICENSE
├─ README.md

│  └─ icon.svg
└─ js/
   ├─ state.js
   ├─ utils.js
   ├─ history.js
   ├─ layers.js
   ├─ canvas-render.js
   ├─ drawing-tools.js
   ├─ image-placement.js
   ├─ selection-tools.js
   ├─ project-storage.js
   ├─ ui.js

   └─ app.js
```

`index.html` ではES Modulesを使わず、グローバルスコープのJavaScriptを `defer` 付きで読み込みます。アプリ表示上のバージョンは `Version 1.0.0` です。一方、`index.html` や `service-worker.js` で使用している `v26` などの番号は、PWAキャッシュ更新用の内部番号です。

## PWA更新が反映されない場合

GitHub PagesやPWAでは、Service Workerやブラウザキャッシュにより古いHTML / CSS / JavaScript / manifestが残る場合があります。更新が反映されない場合は、以下を試してください。

1. ページを数回再読み込みする
2. ブラウザのサイトデータ / キャッシュを削除する
3. ホーム画面に追加したPWAを一度削除する
4. 再度公開URLを開く
5. 必要に応じてPWAを再インストールする

開発時にHTML / CSS / JavaScript / manifestを変更した場合は、`index.html` のCSS / JS読み込みクエリ、`service-worker.js` の `CACHE_NAME`、`APP_SHELL` を同じキャッシュ更新番号にそろえて更新してください。
