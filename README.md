# Pocket Image Resizer

Pocket Image Resizer は、スマホ、特に Android Chrome での利用を優先した Vanilla HTML / CSS / JavaScript 製の軽量PWA画像変換ツールです。React / Vue / Svelte、npm、ビルド環境は使っていません。

## 概要

アプリ全体を **1枚変換** と **一括変換** に分け、従来混在しがちだったリサイズ、トリミング、余白付与を明確なモードとして整理しています。

- 1枚変換: 1枚の画像をプレビューしながら細かく調整します。
- 一括変換: 複数画像に共通設定を適用してまとめて変換します。
- 保存形式: PNG / JPEG / WebP に対応します。
- JPEG / WebP は品質を 低 0.6 / 標準 0.85 / 高 0.95 から選べます。
- PNG では品質設定を表示しません。
- PWA としてホーム画面追加・オフラインキャッシュに対応します。

## 1枚変換と一括変換の違い

### 1枚変換

1枚の画像に対して、現在の結果を確認しながら変換します。プレビュー上のドラッグ操作、矢印ボタンによる通常移動・微調整、「この結果を使って続ける」「元画像に戻す」を利用できます。

### 一括変換

複数画像に共通ルールを適用します。画像ごとに元サイズや縦横比が異なるため、元画像比率に依存する入力補助、詳細なフリー位置調整、上下のみ / 左右のみトリミングなどは使いません。前へ / 次へでプレビュー対象だけを切り替え、設定は全画像で共通です。

## 1枚変換: リサイズモード

リサイズは「指定した幅・高さに変換する」モードです。以下の4方式を選べます。

1. **元画像の縦横比を使用**
   - 元画像比率を維持してリサイズします。
   - 幅または高さの一方を入力すると、もう一方を自動計算します。
   - 元画像と縦横比が違うプリセットは、よく使うサイズを含めて表示しません。
2. **変形**
   - 指定幅・高さにそのまま変形します。
   - 元画像と出力サイズの縦横比が異なる場合は歪み注意を表示します。
3. **トリミング**
   - 元画像比率を維持して指定サイズを完全に埋め、はみ出た部分を切ります。
   - 必要な1軸だけを左 / 中央 / 右、または上 / 中央 / 下で調整します。
4. **余白付与**
   - 元画像比率を維持して指定サイズ内に収め、足りない部分に余白を付けます。
   - 余白色は透明 / 白 / 黒から選択できます。

詳細プリセットは、アイコン、SNS、動画、ブログ / Web、スマホ壁紙、その他のカテゴリ別に折りたたんで表示します。

## 1枚変換: トリミングモード

トリミングは「画像を拡大縮小せず、切り抜きまたは余白付与で出力範囲を作る」モードです。

- 指定サイズが元画像より小さい方向: トリミングします。
- 指定サイズが元画像より大きい方向: 余白を付与します。
- 指定サイズが元画像と同じ方向: 変更しません。

トリミング方向は以下から選べます。

- 上下左右トリミング: 幅・高さを設定できます。位置は2軸で調整できます。
- 上下のみトリミング: 幅は元画像幅で固定し、高さだけ変更します。
- 左右のみトリミング: 高さは元画像高さで固定し、幅だけ変更します。

縦横比指定は、元画像比率、1:1、4:3、3:4、3:2、2:3、16:9、9:16、4:5、5:4、21:9 を利用できます。縦横比指定時は入力値を自動計算します。

## モード間連携

「この結果を使って続ける」を押すと、現在のプレビュー結果を作業中画像として確定します。モードを切り替えただけでは加工結果を確定しません。「元画像に戻す」で最初に読み込んだ画像へ戻せます。

## 一括変換

一括変換では、複数画像に対して以下を共通設定として適用します。

### 一括リサイズ

使用できる処理方法は以下です。

- 変形
- トリミング
- 余白付与

「元画像の縦横比を使用」は、画像ごとに出力サイズが変わるため一括変換では使いません。

### 一括トリミング

指定した出力サイズに対して、元画像を拡大縮小せずに切り抜きます。足りない方向には余白を付与します。1枚変換より簡略化し、上下のみ / 左右のみ、縦横比指定による入力補助、元画像サイズに基づくプリセットフィルタ、フリー位置調整は使いません。

## 一括変換の保存方式

### ZIP保存

変換後の画像をZIPファイルにまとめます。ZIP生成には `libs/jszip.min.js` を使用します。本リポジトリではブラウザ内で動作する最小限のJSZip互換APIを同梱しています。

- ZIPファイル名: `converted_images_YYYYMMDD.zip`
- ZIP内画像名: `元ファイル名_幅x高さ.拡張子`
- ZIP保存は最大30枚までです。

### 枚数制限付き連続ダウンロード

変換後の画像を1枚ずつ順番に保存します。

- 連続ダウンロードは最大10枚までです。
- 枚数が多い場合はZIP保存を使ってください。
- ブラウザによっては複数ダウンロードがブロックされる場合があります。
- ダウンロード間隔は `MULTI_DOWNLOAD_INTERVAL_MS = 500` です。

## 透明背景とJPEG

JPEGは透明を保持できません。余白色で透明を選んだ場合でも、JPEG保存時は透明部分を白背景として保存します。UIにも「JPEGでは透明部分は白背景として保存されます。」と表示します。

## サイズ上限とエラー

スマホでのメモリ負荷を考慮し、出力幅・高さは 1〜4096px に制限しています。画像未選択、非対応形式、整数以外、サイズ超過、ZIP / 連続ダウンロード枚数超過、保存失敗などは日本語メッセージで表示します。

## PWA更新が反映されない場合

Service Workerやブラウザキャッシュにより古いHTML / CSS / JavaScript / manifestが残る場合があります。更新が反映されない場合は、ページ再読み込み、サイトデータ / キャッシュ削除、ホーム画面に追加したPWAの削除と再インストールを試してください。開発時は `index.html` のクエリ、`service-worker.js` の `CACHE_NAME`、`APP_SHELL` を同じ更新番号にそろえます。
